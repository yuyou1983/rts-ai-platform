关键架构决策 (ADR)

Architecture Decision Record — 三项核心决策
ADR-001
渐进式拆分
运行时 Agent 不在 M0 阶段拆分，而是随里程碑渐进增加。M0 单体 → M1 三核心 → M2 全 Agent。
✓ 降低初期复杂度，避免过早抽象；每个阶段交付可验证闭环
ADR-002
协议隔离
AgentHub 接口按版本演进 (v0 → v1 → v2)，各阶段 Agent 使用对应版本协议，互不干扰。
✓ 协议版本化保证向后兼容，M0 Agent 可在 M1+ 继续运行
ADR-003
LLM 不进高频环
LLM 推理仅用于宏观战略 (1-2 Hz) 和离线分析，高频决策 (10-20 Hz) 使用行为树/规则引擎。
✓ 延迟约束: LLM 推理 50-200ms 不可用于微操；高频环保持确定性
已采纳 已采纳 已采纳 三项决策均通过技术委员会评审
