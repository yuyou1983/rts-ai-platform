# M0 基线 Agent

## 设计原则

1. **单体架构**: M0 不拆分 Agent，所有决策在一个状态机内完成
2. **规则驱动**: 可配置的规则表 + 优先级仲裁
3. **可观测**: 每次决策通过 obs.world.v1 观察全局状态
4. **可回放**: 所有命令通过 cmd.micro.v1 下发，确保回放可重现

## 状态机

```
     ┌──────────┐
     │  OPENING │ ← 初始状态
     └────┬─────┘
          │ 满足基本经济阈值
          ▼
     ┌──────────┐
     │ ECONOMY  │ ← 采集 + 建造循环
     └────┬─────┘
          │ 供给充足 + 有军事单位
          ▼
   ┌────────────┐
   │  MILITARY   │ ← 进攻/防守判断
   ├──────┬─────┤
   │ATTACK│DEFEND│ ← 子状态切换
   └──────┴─────┘
          │ 资源枯竭或胜负
          ▼
     ┌──────────┐
     │ END_GAME │ ← 全力一搏或防守
     └──────────┘
```

## 规则优先级

1. 战斗安全: 基地被攻击 → 立即防守
2. 供给阻塞: 供给不足 → 优先建造补给建筑
3. 经济阈值: 空闲工人 > 1 → 分配工人
4. 军事扩张: 有余力 → 生产军事单位
5. 进攻时机: 军事单位数 ≥ 阈值 → 进攻

## 配置参数 (Tuning Knobs)

| 参数 | 默认值 | 安全范围 | 说明 |
|------|--------|----------|------|
| opening_workers | 12 | 8-16 | 开局工人目标 |
| attack_threshold | 8 | 6-20 | 进攻最低单位数 |
| defend_radius | 15.0 | 8-25 | 防守响应半径 |
| expand_minerals | 400 | 300-600 | 扩张最低矿物 |
| idle_worker_limit | 1 | 0-3 | 允许的最大空闲工人 |
| supply_buffer | 4 | 2-8 | 供给缓冲提前量 |

## 接口契约

输入: obs.world.v1 / 输出: cmd.micro.v1
